# DARLEK CANN: GIT ARCHITECTURE MANIFEST

## Purpose
This document defines the exclusion policy for the DARLEK CANN v3.0 ecosystem. It ensures that the repository remains a clean, reproducible codebase while isolating volatile AGI state, simulation weights, and sensitive credentials.

## Exclusion Tiers
1. **Hardened Security**: Cryptographic keys and environment secrets are strictly excluded. Only templates are permitted.
2. **Simulation Integrity**: Binary weights, model checkpoints, and training data are excluded to prevent repository bloat and ensure clean CI/CD pipelines.
3. **Agent State**: Local agent memory, swarm logs, and evolution history are ignored to prevent local machine state from polluting the global repository.

## Integration Guidelines
- When adding new simulation modules, ensure the output directory is added to the `.gitignore` immediately.
- Never commit `.env` files. Use `.env.example` for schema definitions.
- All self-refactoring logs generated by the DARLEK CANN engine must be directed to `.evolution_history/`.
